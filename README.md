#  Advanced UI & Custom Components (Flutter)

This repository contains a collection of **reusable and customizable UI components** built with Flutter. It is part of a structured learning module focused on creating scalable, maintainable, and production-ready UI systems.

The project is divided into multiple mini-apps, each demonstrating a specific category of UI components.

---

## Project Structure

```
Month 3: Advanced UI & Custom Features
│
├── custom_buttons_app
├── custom_inputs_app
├── custom_cards_app
└── modals_app
```

Each module is self-contained and demonstrates best practices for reusable widget design.

---

##  Features Overview

###  Custom Buttons

* Reusable `CustomButton` component
* Multiple variants:

  * Primary Button
  * Secondary Button
  * Outline Button
  * Text Button
* Icon support
* Loading state with `CircularProgressIndicator`
* Ripple and hover effects
* Button animations
* Fully customizable styles using `ButtonStyle`

---

###  Custom Input Fields

* Reusable `CustomTextField`
* Built-in validation support
* Specialized fields:

  * Password Field (visibility toggle)
  * Search Field (clear button)
  * OTP Field (6-digit input system)
* Prefix & suffix icons
* Error handling and display
* Consistent styling across all inputs

---

###  Custom Cards

* Base reusable `BaseCard` component
* Multiple card types:

  * Product Card
  * User Card
  * News Card
  * Stats Card
* Features:

  * Shadows and elevation
  * Gradient backgrounds
  * Tap animations
  * Swipe-to-delete gesture
  * Shimmer loading effect
* Fully customizable properties

---

###  Bottom Sheets & Dialogs

* Custom modal implementations:

  * Simple Bottom Sheet
  * Scrollable Bottom Sheet
  * Modal Bottom Sheet
* Dialog types:

  * Alert Dialog
  * Confirmation Dialog
  * Custom Form Dialog
* Additional features:

  * Filter Bottom Sheet (e-commerce style)
  * Popup menus
  * Result handling using `Navigator.pop()`
  * Consistent UI styling

---

##  Key Concepts Covered

* Reusable widget architecture
* Separation of concerns
* Stateless vs Stateful design decisions
* UI consistency and theming
* Flutter animations and gestures
* Form validation and input handling
* Scalable component design
